Workflow

End-to-End Process

The agent handles the full lifecycle in two phases, with a human review gate in between.

PHASE 1 Draft the Agreement

1. Read Gmail Thread

Search for the influencer's email thread in affiliates@onzenna.com or affiliates@zezebaebae.com. Extract all deal terms: name, handle, deliverables, payment, platforms, whitelisting terms.

Gmail MCP Python Script (zezebaebae)

2. Fill the .docx Template

Read the agreement template from the NAS, replace all 14 red placeholders with extracted data. Apply W9 and whitelisting section logic automatically.

python-docx Synology NAS

3. Save Draft

Save as [@handle] Influencers agreement_paid.docx in the Agreements folder on the NAS.

NAS File System

4. Discord Notification

Send a summary to the Discord channel: influencer name, payment amount, and deliverables. Ask the user to review and type "진행" to continue.

Discord Webhook

HUMAN REVIEW GATE

The agent stops here and waits. The user reviews the .docx draft and types "진행", "confirm", or "continue" in the terminal to proceed.

PHASE 2 Upload to DocuSeal

5. Create DocuSeal Template

Encode the .docx as base64, upload via DocuSeal API. Template name: @handle + Full Name. Signature fields are auto-detected.

DocuSeal MCP

6. Final Discord Notification

Notify the user that the DocuSeal document is ready. The user then manually adds recipients and sends in the DocuSeal web UI.

Logic

Business Rules

Two conditional rules that adjust the agreement content automatically.

$ W9 Tax Form Rule

If total payment is $600 or more, the W9 requirement sentence is included. Below $600, the entire sentence is removed from the document — not left blank.

& Whitelisting Section Logic

The template has two versions of Section 5. If the deal includes whitelisting or Spark ads, the first version is kept with the duration filled in. Otherwise, only the second (generic) version is kept. The unused section is deleted entirely.

Integrations

Gmail MCP

Reads influencer email threads from affiliates@onzenna.com. Falls back to a Python script for the zezebaebae inbox.

Synology NAS

Reads the .docx template and saves completed drafts to /Volumes/Orbiters/Influencer Team/Agreements/.

Discord Webhook

Sends status notifications at two checkpoints: after draft creation and after DocuSeal upload.

Safety

Human-in-the-Loop

The agent is designed to never send a document without human approval.

Review Gate

After drafting, the agent stops and waits for explicit confirmation ("진행") before uploading to DocuSeal.

Final Send is Manual

The agent never calls send_documents. Adding recipients and sending is always done by a human in the DocuSeal web UI.
